# A股板块波段强度评分设计

**目标**

新增一个独立于现有日报链路的命令行功能。用户输入板块名称或板块代码后，程序基于最新候选池和公开市场数据，计算板块波段强度评分，输出排名表、指标明细、龙头候选、风险提示、操作建议和可复用 JSON。

**边界**

- 不改原有 `ingest`、`report`、`run-once` 的行为。
- 新功能放在 `app/sector_strength/`，通过新 CLI 子命令调用。
- 候选池优先使用现有 SQLite 中最近一次抓取的前 `50` 个板块。
- 若用户输入板块不在候选池内，尝试按公开板块列表解析后追加分析，但会标注“不在候选池”。

**数据源策略**

- 第一优先级：AkShare
- 第二优先级：东方财富公开接口
- 候选池兜底：现有 SQLite 快照

**候选池策略**

- 默认候选池 = `概念板块 + 行业板块`
- 概念板块优先读取最近交易日的 `1日榜` 前 `N`
- 概念来源优先级：`tonghuashun` > `eastmoney`
- 若概念 `1日榜` 不可用，则回退到 `5日榜`
- 行业板块默认实时抓取前 `N`
- 若本地概念快照为空，则实时抓取概念资金流榜单作为兜底候选池

**模块拆分**

- `app/sector_strength/data_fetcher.py`
  - 板块名录解析
  - 板块 60 日 K 线抓取
  - 沪深 300 60 日 K 线抓取
  - 板块 5 日、10 日主力净流入抓取
  - 成分股、涨停池抓取
  - 数据源回退和字段标准化
- `app/sector_strength/indicator_calculator.py`
  - 5 日、20 日涨幅
  - RS、均线、广度、活跃度、龙头候选等指标计算
- `app/sector_strength/scoring_model.py`
  - 六大维度子分
  - 风险惩罚
  - 总分和操作建议
- `app/sector_strength/main.py`
  - 候选池装载
  - 输入匹配
  - 并发抓数
  - 汇总输出、保存 JSON / 文本摘要

**输出结构**

- 终端：
  - 候选池摘要
  - 评分排名前列
  - 用户输入板块的详细结果
  - 输出文件路径
- 文件：
  - `output/sector-strength/<trade_date>/result.json`
  - `output/sector-strength/<trade_date>/summary.txt`

**缺失数据处理**

- 原始缺失字段保留 `null`
- 子指标缺失时不补造假值
- 总分按“可用子分权重归一化后再减风险惩罚”计算
- 同时输出 `missing_metrics` 和 `warnings`

**评分模型**

- 资金持续性：看 `5日`、`10日` 主力净流入占成交额比例与连续性
- 相对强度 RS：看 `RS_5日`、`RS_20日`
- 趋势结构：看 `MA5/10/20/60` 与均线多头关系
- 板块广度：看上涨家数占比、`>3%`、`>5%`、涨停密度
- 成交活跃度：看今日成交额 / `20日` 均成交额
- 龙头质量：取成交额 Top5 中涨幅最高股票，参考涨幅、换手率、是否涨停
- 风险惩罚：看跌破中期均线、RS 转弱、资金转负、广度不足、缺失数据过多

**验证方式**

- Python 语法编译检查
- 新 CLI 帮助命令检查
- 真实板块样例运行，确认能输出文本和 JSON

**当前实现说明**

- 行业板块已默认纳入候选池
- `candidate_limit` 当前按“每类候选池数量”理解，因此总分析样本通常会大于单个 `candidate_limit`
