# 基金排行榜独立报告设计

**日期**: 2026-04-26

## 目标

新增一个独立于现有日报的“基金排行榜报告”能力。

第一阶段先完成：

- 抓取天天基金 `今日 / 近一周 / 近一月 / 近三月 / 近六月 / 近一年` 前 `150`
- 按 `A/C` 规则做阶段内去重
- 统计基金跨阶段重复出现率
- 生成独立 `HTML / JSON / summary` 输出

第二阶段再补：

- 基金前十大重仓股抓取
- AI 大板块 / 细分板块识别
- 板块聚合排行与热度分析

## 产品形态

不改现有 `run-once` 主日报。

新增独立命令：

```bash
python3 -m app.cli fund-rank-report
```

独立输出目录：

- `output/fund-rank/<trade_date>/report.html`
- `output/fund-rank/<trade_date>/result.json`
- `output/fund-rank/<trade_date>/summary.txt`

## 数据范围

### 排行榜阶段

- `day`
- `week`
- `month`
- `quarter`
- `half_year`
- `year`

每个阶段抓取前 `150`。

### 去重规则

同一基础基金名下：

- 有 `C` 类时保留 `C`
- 只有 `A` 类时保留 `A`
- 其他份额类型不做扩写兼容，按原名保留

“同一基础基金”仅做最小归一：

- 去掉基金名尾部 `A` / `C`
- 去掉基金名尾部 `A类` / `C类`

不额外猜测其他后缀语义。

## 第一阶段输出

### 第一部分：分阶段基金排行榜

按 6 个阶段分别展示：

- 排名
- 基金名称
- 基金代码
- 对应阶段涨幅
- 是否为 `C` 优先保留结果

### 第二部分：高频重复基金榜

按基金维度聚合，展示：

- 基金名称
- 基金代码
- 出现阶段
- 出现次数
- 重复出现率
- 各阶段排名
- 各阶段涨幅

排序逻辑：

1. 出现次数降序
2. 最佳排名升序
3. 阶段涨幅均值降序
4. 基金名称升序

### 颜色标识

- `1 次`: 普通
- `2 次`: 轻度活跃
- `3 次`: 中度活跃
- `4 次`: 高度活跃
- `5-6 次`: 强势高频

颜色只表达重复程度，不表达涨跌方向。

## 第二阶段输出

### 基金重仓股

对进入报告分析范围的基金抓取“最新公开披露前十大重仓股”：

- 股票名称
- 股票代码
- 持仓占比
- 披露期

### AI 板块识别

AI 负责直接输出：

- 大板块
- 细分板块
- 判断依据
- 置信度

规则层只负责整理输入、缓存输出，不预设固定板块枚举。

### 板块聚合排行

支持：

- 全部基金的大板块排行
- 全部基金的细分板块排行
- 分阶段板块排行
- 高频重复基金子集对应的板块排行

板块排行字段：

- 板块名称
- 涉及基金数量
- 涉及基金列表
- 平均涨幅
- 代表基金
- 主要代表持仓股
- 板块热度

## 数据流

第一阶段：

1. 抓取 6 个阶段原始排行
2. 解析为统一记录
3. 阶段内按 `A/C` 规则去重
4. 生成跨阶段基金聚合结果
5. 渲染独立报告并输出 JSON / summary

第二阶段：

1. 基于第一阶段唯一基金集合抓前十大重仓股
2. 调 AI 输出板块识别结果
3. 聚合板块排行
4. 在独立报告追加基金板块识别区和板块聚合区

## 降级策略

- 单个阶段抓取失败：保留其他阶段，页面顶部标告警
- 全部阶段失败：命令返回失败
- 第二阶段持仓抓取失败：第一阶段结果仍可输出
- 第二阶段 AI 失败：持仓信息照常展示，板块区显示告警
